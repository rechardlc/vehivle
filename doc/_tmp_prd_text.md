说明：本文件为单一正式版产品需求文档，适用于项目评审、研发排期、外包报价、测试验收与上线前检查。
1. 项目概述
项目背景：渠道商（总代/批发商）目前依赖微信群发图片、语音报单等传统方式向下游门店分发车型信息，效率低下且容易出错。
核心目标：打造一个无需登录的纯展示类“数字化电子画册”，帮助下游门店极速看车、查参数，并可直接向进店散客展示，形成轻量级线上展厅。
目标用户：下游二级经销商、门店老板，以及门店接待的进店散客。
产品原则：极简、快速加载、参数直观、强引导联系；不包含交易、支付和真实底价，不做复杂学习成本。
核心价值：将原本分散在微信群、相册、语音和 Excel 中的车型素材收口到一个可统一展示、统一管理、统一分享的小程序中。
2. 版本目标与范围边界
2.1 V1.0 产品定位
V1.0 是一个“展示型数字展厅 MVP”，目标是先解决“看车资料不统一、参数传递不标准、联系路径不顺畅”的问题，而不是一步到位做成在线订货平台。
2.2 默认产品假设
按单渠道商单后台方式设计，不做多租户 SaaS 能力。
后台主要由渠道商老板或 1-3 名运营人员使用。
V1 不做下游客户登录、审核、分级、专属报价、订单、支付、库存同步。
用户主要通过扫码、转发链接或首页分类浏览进入，不依赖站内关键词搜索。
V1 先做埋点预埋，不强依赖完整数据看板。

2.4 V1.0 成功标准
渠道商可在无技术支持前提下完成车型资料的新增、编辑、上架、下架和首页运营位配置。
下游用户可以在 3 次以内点击路径中到达目标车型详情页，并可一键拨号或复制微信联系。
车型参数展示方式统一，避免继续依赖大图海报传参。
旧分享链接在车型下架后有明确兜底提示，不出现空白页或异常跳转。
3. 核心业务流程
3.1 渠道商发布流程
登录后台 -> 创建/维护分类 -> 新建车型 -> 录入基础信息与参数 -> 上传图片/视频 -> 选择首页运营标签 -> 预览 -> 上架发布。
3.2 下游客户浏览流程
扫码/分享链接进入小程序 -> 通过首页运营区或分类库浏览车型 -> 查看图片、卖点和参数 -> 点击拨号或微信联系 -> 转发给同行或顾客。
3.3 车型维护与下架流程
渠道商在后台修改车型资料或执行下架 -> 小程序前台实时同步最新状态 -> 旧链接访问时展示“该车型已下架/暂不可查看”页面，并提供返回首页或联系上游入口。
3.4 用户端浏览路径图
说明：为便于研发与测试快速理解前台路径，V1 用户端浏览流程建议按以下路径实现。
扫码 / 分享链接 / 首页访问
↓
首页 Banner / 分类金刚区 / 运营专区入口
↓
分类/品牌筛选 + 车型列表（触底加载 / 下拉刷新）
↓
车型详情（媒体、参数、价格口径、免责声明）
↓
拨号联系 / 微信联系 / 原生分享
3.5 后台车型状态流转图
说明：V1 推荐采用清晰的内容状态模型，减少前后台展示口径不一致。
草稿  →  已上架  →  已下架  →  逻辑删除
状态说明：
• 草稿：后台可编辑、可预览，前台不可见。
• 已上架：前台可见，可进入首页、分类页、运营专区和分享链接。
• 已下架：前台不可见；旧链接访问时统一进入下架兜底页。
• 逻辑删除：默认不在后台常规列表中展示，必要时仅管理员可恢复或审计。

4.1 小程序端关键业务规则
价格规则：V1 由后台统一控制前端展示“建议零售价”或“电话询价”；若车型未填写价格，则即便全局开启价格展示，也统一回落为“电话询价”。
联系规则：电话按钮点击后直接拉起系统拨号；微信按钮点击后展示客服二维码、微信号和“复制微信号”按钮。
标签规则：一个车型可同时属于多个首页运营区块；区块顺序固定为“最新到货 -> 特价清仓 -> 热销爆款”。
展示规则：车型一旦下架，首页、列表页、运营专区、分享链接均不可继续看到正常详情内容。
分享规则：V1 不做分享海报生成器，只调用平台原生分享能力，保证实现简单稳定。

6.5 关键业务规则补充
分类规则：V1 最多两级分类，一级用于车型大类，二级用于品牌/系列，不再继续嵌套。
内容状态规则：车型仅在“已上架”状态下对前台可见；草稿和已下架均不可见。
删除规则：删除建议采用逻辑删除；已删除内容默认不在前台和后台列表中显示。
首页专区规则：若某个专区没有可展示内容，则整个专区模块隐藏，不显示空白标题。
参数展示规则：空值字段默认隐藏，不展示“-”占位；若整张参数表为空，则给出统一提示语。
分享规则：分享路径必须可直达当前车型详情；车型下架后，旧分享链接统一落至兜底页。
联系规则：若仅配置电话未配置微信，则只展示电话按钮；反之亦然。
价格规则：V1 严禁出现渠道底价、拿货价、优惠券、支付金额等交易型价格表达。

9. 产品未来迭代演进路线图（Roadmap）
考虑到 B2B 渠道分销的实际业务落地节奏，后续产品将围绕“权限与价格管理”“在线订货协同”“数据与营销赋能”三个维度进行阶段性升级。
9.1 迭代一：V2.0 专属报价与渠道客户体系（核心壁垒期）
目标：从“公开画册”升级为“渠道专属的数字化进货平台”，实现下游客户身份认证与“一客一价”。
新增下游客户注册与认证：引入微信授权登录/手机号验证码登录；新用户提交门店名称、联系人、营业执照（可选）进行入驻申请。
新增千人千面报价系统：后台可对下游客户进行分级打标，设置裸车底价和电池组合差价；前端登录后自动展示所属级别的专属拿货总价。
新增“展厅模式”一键切换：下游门店向散客展示时，可隐藏渠道拿货底价，改为显示官方指导零售价。
9.2 迭代二：V3.0 在线订货与库存协同（交易闭环期）
目标：取代微信群语音报单、手写单据，降低错漏率，实现订货流程线上化。
新增灵活选配与进货单：支持选择裸车或不同规格电池组合，系统实时计算总批发价。
新增非支付型订单流转：前端提交进货单后生成意向订单，后台执行“待确认 -> 已收款 -> 已发车 -> 已完结”状态流转。
新增前置库存状态提示：后台可打上“现货充足”“仅剩少量”“预定中-预计 X 号到货”等库存标签。
9.3 迭代三：V4.0 营销赋能与数据决策（增长爆发期）
目标：变被动接单为主动营销，利用数据指导进货与清仓。
新增特价清仓与限时特供区：将滞销车型、尾货或促销车型集中投放，快速回笼资金。
新增渠道政策与公告推送：通过首页弹窗或消息中心触达下游门店老板。
新增渠道商数据看板：统计浏览次数、转发次数、客户活跃度和流失预警，反向指导拿货与拜访。
