# PRD 与技术映射

## 1. 接口域映射
- 认证域：`/api/v1/admin/auth/login`、`/refresh`、`/me`。
- 车型域：列表、创建草稿、发布、下架、批量状态、预览、复制。
- 分类域：分类 CRUD 与排序、启停。
- 参数模板域：模板与参数项维护。
- 媒体域：上传签名、上传完成回写。
- 系统设置域：获取与更新全局配置。
- 公开域：首页聚合、分类选车、分享落地校验。

## 2. 数据表映射
- admin_users -> 认证与权限。
- categories -> 首页分类、选车筛选。
- param_templates + items -> 详情参数结构。
- vehicles + param_values -> 核心车型信息。
- media_assets -> 图片/视频元数据。
- system_settings -> 全局价格模式与免责声明。
- audit_logs -> 关键操作审计。

## 3. 核心业务规则映射
- 车型仅 `published` 对前台可见。
- 下架后分享链接返回兜底信息。
- 全局价格模式影响列表与详情统一口径。
- 联系方式缺失时隐藏对应按钮。
- 空参数字段不展示，整表为空时给出提示。

## 4. 非功能映射
- 性能目标：首页首屏 <= 3s，详情核心内容 <= 2s。
- 可维护性：目录分层、统一错误码、统一响应结构。
- 可运维性：结构化日志、请求链路 ID、部署分环境。

## 5. 验收映射建议
每个业务域至少准备：
- 正常流 1 组。
- 参数异常 1 组。
- 权限异常 1 组。
- 状态冲突 1 组。
并将结果回填到测试文档与验收清单。
