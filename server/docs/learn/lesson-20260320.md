## 今日完成

1. **enum 落地**：`internal/domain/enum/enum.go`
   - `VehicleStatus`：draft / published / unpublished / deleted（与 PRD 状态机对齐）
   - `PriceMode`：show_price / phone_inquiry

2. **model 落地**：`internal/domain/model/vehicle.go`
   - `Vehicle` 结构体：ID、CategoryID、Name、CoverMediaID、PriceMode、MSRPPrice、Status、SellingPoints、SortOrder、CreatedAt、UpdatedAt
   - `Publish()`、`Unpublish()` 方法：状态变更 + UpdatedAt 更新

3. **rule 落地**：`internal/domain/rule/vehicle_publish.go`
   - `PublishRequirements` 结构体：HasCoverImage、HasDetailImages、HasRequiredParams
   - `CanPublishVehicle(v, requirements)`：纯函数，校验草稿状态、必填项、封面图、详情图、参数完整性

4. **PostgreSQL 基础设施**：`internal/infrastructure/postgres/postgres.go`
   - `Open(cfg, log)`：`gorm.Open`、从配置设置 `MaxOpenConns` / `MaxIdleConns` / `ConnMaxLifetime`；连接成功日志中 DSN 脱敏（如 `***`）
   - `Ping(ctx, db, skip)`：启动或就绪探活；`Close(db)`：进程退出时释放连接池

5. **bootstrap 装配**：`internal/bootstrap/bootstrap.go`
   - `pgsqlConnPool()` → `postgres.Ping` → `router.New(r, b.logger, b.db).Register()`（先连库再挂路由）

6. **router**：`internal/transport/http/router/router.go`
   - `New(engine, logger, db *gorm.DB)` 注入数据库
   - `Register`：`userHandler := handler.NewUser(r.db)`、`vehiclesHandler := handler.NewVehicles(r.db)`，各路由绑定方法值（避免每条路由重复 `New*`）
   - `admin/user`、`admin/vehicles`、`public/vehicles`（GET）

7. **handler（构造函数注入）**
   - `handler/user.go`：`User` + `NewUser(db)`，方法 `Get`、`Create`、`Update`、`Delete`（占位）
   - `handler/vehicles.go`：`Vehicles` + `NewVehicles(db)`，方法 `List`、`Create`、`Update`、`Delete`（占位）

8. **Makefile（server 根目录）**：统一工程命令，与 `configs.Load()`、`VEHIVLE_APP_ENV`、`.air.toml` 对齐
   - 常用：`make help`（默认）、`make run`（设置 `VEHIVLE_APP_ENV` 后 `go run ./cmd/api`）、`make dev`（`air`）、`make build`（输出 `tmp/main` 或 `tmp/main.exe`）、`make check`（`fmt` + `vet` + `test`）、`make clean`
   - 依赖与整理：`make deps`、`make tidy`
   - 仓库根目录可执行：`make -C server <target>`（需本机已安装 GNU Make）
   - Windows 使用 `cmd` 兼容写法（`set VAR=...&&`、`mkdir`/`rmdir`）；类 Unix 使用 POSIX 写法

## 复盘

- domain 层与业务逻辑解耦，enum/model/rule 可独立于 DB 测试
- `CanPublishVehicle` 返回 `(ok bool, errs []string)`，便于 handler 组装错误信息
- **企业常见写法**：Handler 为 struct，`New*` 注入 `*gorm.DB`（后续可改为注入 Repository 接口）；Router 内每个资源组只 `New` 一次 handler 再绑路由
- 连接层：`bootstrap` 编排顺序清晰；日志不落明文 DSN
- `public/vehicles` 若写 `handler.NewVehicles(r.db).List` 会与 admin 组各建一次实例，可改为复用同一 `vehiclesHandler.List`
- handler 占位就绪，待迁移与 Repository 后实现真实逻辑
- Makefile 将「跑服务、热重载、编译、检查、清缓存」收口为固定命令；未安装 `make` 时需自行安装 GNU Make 或使用等价命令
