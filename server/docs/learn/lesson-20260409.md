## 一、为什么要 `media_assets`（和 GC 的关系）

- **主目的**：在 PostgreSQL 里为「每次成功上传到 OSS 的文件」立一份**档案**：`storage_key`（桶内对象键）、`mime_type`、`file_size`、`asset_type`、`created_at`；并生成稳定主键 **`id`（UUID）**。
- **业务引用**：`vehicles.cover_media_id` 存 **`media_assets.id`**，而不是裸路径字符串——换 CDN 域名、改桶策略时，业务侧只认 id，展示 URL 由服务端用 `storage_key` + 配置拼接。
- **GC**：有集中档案后，更容易做「无引用删除 / 过期清理」；但 **GC 只是附带收益**，不是建表的唯一理由。

## 二、迁移 `000003_create_media_assets_table`

- 文件：`migrations/000003_create_media_assets_table.up.sql` / `.down.sql`
- 核心：`CREATE TABLE media_assets (...)`；**`UNIQUE(storage_key)`**，避免同一对象键重复登记（与上传生成唯一键策略一致）。
- 执行：`server/` 目录下 `go run ./cmd/migrate -op up`（见 [migrations/README.md](../migrations/README.md)）。

## 五、`MinioClient.ObjectPublicURL`

- 位置：`infrastructure/oss/client.go`
- 规则：与上传成功响应中的公网地址一致：`PublicURL + "/" + Bucket + "/" + storageKey`
- 用途：车型 **`List`/`Create`/`Update`/`Duplicate` 返回前**，根据 `cover_media_id` 查 `storage_key`，写入 **`coverImageUrl`**（JSON 字段名 `coverImageUrl`）。

## 六、`VehicleService` 扩展

- **`GetById`**：供 Handler 发布前取车、避免全表扫。
- **`VehicleUpdateInput` + `UpdateVehicle`**：部分更新（指针字段语义对齐分类 PATCH，见 [lesson-20260325.md](./lesson-20260325.md)）。
- **`SoftDelete`**：`status = deleted`（逻辑删除）。
- **`Publish` / `Unpublish`**：沿用 `rule.CanPublishVehicle`；MVP 对详情图/参数项要求可放宽，避免阻塞发布（产品后续收紧）。
- **`Duplicate`**：新 UUID 草稿，复制封面 id 等字段。
- **`BatchSetStatus`**：批量 `published` / `unpublished`（跳过已删除）。

---

## 七、`Vehicles` Handler 与路由

- **`attachCoverImageURLs`**：`List`/`Create`/`Update`/`Duplicate` 返回前批量补 `CoverImageURL`。
- **路由**（需在 **`/:vehicle_id`** 静态子路由之前注册字面量路径）：
  - `POST /admin/vehicles/batch-status`
  - `POST /admin/vehicles/:vehicle_id/publish|unpublish|duplicate`
- **`NewVehicles(db, oss)`**：注入 OSS 用于拼 URL；公开 `GET /public/vehicles` 与后台共用 `List` 逻辑（`onlyPublished` 由路径判断）。

## 八、前端（admin）契约

- `mediaApi.uploadImage` → **`ImageUploadResult { id, url, storageKey }`**，表单 **`coverMediaId` = `id`**。
- `Vehicle.coverImageUrl`：列表/编辑回显；**`ImageUploader`** 用 **`previewFromServer`** 在仅有 UUID 时显示缩略图。

## 九、验收提示

1. 迁移应用到当前 DSN：`go run ./cmd/migrate -op version` 与迁移文件最大版本一致。
2. API 与 migrate **同一 `VEHIVLE_DATABASE_DSN`**，否则会出现「migrate 已 version=3 但上传仍缺表」的错觉。
3. 旧数据若 `cover_media_id` 仍为历史「对象键」字符串，需数据修复或重新上传封面。
