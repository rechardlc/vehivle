## 今日完成

1. **bootstrap 落地**：`internal/bootstrap/bootstrap.go`
   - `Bootstrap` 结构体持有 `cfg`、`logger`
   - `New(cfg)` 创建实例
   - `Run()` 返回 `*gin.Engine`，内部完成：Gin 初始化、中间件链、`router.New(r, b.logger).Register()`
   - 健康检查已移入 router，bootstrap 不再直接注册

2. **main 精简**：`cmd/api/main.go`
   - `configs.Load()` → `bootstrap.New(cfg)` → `b.Run()` → `r.Run(":" + strconv.Itoa(cfg.App.Port))`
   - 入口只做配置加载与启动，业务逻辑集中在 bootstrap

3. **router 落地**：`internal/transport/http/router/router.go`
   - `Router` 结构体持有 `engine`、`logger`
   - `New(engine, logger)` 创建实例，便于后续注入 handler 等依赖
   - `Register()` 注册：`GET /health`、`/api/v1/admin`、`/api/v1/public` 分组
   - `healthHandler` 作为 Router 方法，使用 logger 记录、response.Success 返回
   - **NoRoute/NoMethod 通配**：未注册路由返回统一 JSON 404，方法不匹配返回 405

4. **路由分组**：
   - `GET /health`：健康检查（探活、负载均衡）
   - `GET /api/v1/admin/user/:user_id`：管理端用户详情（handler.UserHandler）
   - `GET /api/v1/public/vehicles`：小程序端占位（匿名 handler）

5. **handler 落地**：`internal/transport/http/handler/user.go`
   - UserHandler、UserListHandler、UserCreateHandler、UserUpdateHandler、UserDeleteHandler
   - admin/user 已挂载 UserHandler，其余待路由注册与 service 接入

6. **response 扩展**：`pkg/response/response.go`
   - `CodeNotFound`、`CodeMethodNotAllowed`
   - `FailNotFound(c, message)`：HTTP 404 + 统一 JSON
   - `FailMethodNotAllowed(c, message)`：HTTP 405 + 统一 JSON

7. **configs 已就绪**：`configs/conf.go`
   - Conf 结构（App、Database、Redis、Oss、JWT）
   - Load、Validate、环境变量覆盖

8. **Air 热更新**：`server/.air.toml`
   - 安装：`go install github.com/air-verse/air@latest`（cosmtrek/air 已迁移至 air-verse）
   - 构建命令修正：`go build -o ./tmp/main.exe ./cmd/api`（入口在 cmd/api，不能用 `.`）
   - 开发时在 `server/` 目录执行 `air`，改代码自动重编译并重启
