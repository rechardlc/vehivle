# Go 后端学习与项目落地路线（前端开发者版）

## 目标
通过“边学边做”的方式，从 Go 零散知识进阶到可独立完成本项目 V1 后端，并具备企业开发基本能力。

## 总原则
- 只学当前阶段会用到的知识，避免信息过载。
- 每学一块就落到本项目真实需求。
- 先跑通主链路，再补边界和工程化。
- 所有代码手写，你只把这份文档当路线图和检查清单。

## 阶段 0：准备期（1-2 天）
### 学习目标
搭好开发环境，理解项目边界和后端职责。

### 你要完成
1. 通读 `doc/tech.md` 与 PRD，明确 V1 必做/不做范围。
2. 确认本 `server` 目录结构与每层职责。
3. 安装 Go、PostgreSQL、Redis，准备 API 调试工具。
4. 写出你自己的“术语对照表”：前端概念 vs 后端概念（路由、事务、鉴权、迁移、幂等）。

### 验收标准
- 你能口述项目核心业务流程（发布、浏览、下架兜底）。
- 你能解释 Handler/Service/Repository/Domain 的区别。

## 阶段 1：Go 基础（3-5 天）
### 学习目标
掌握写后端最常用的 Go 语法与工程组织方式。

### 必学知识
1. 包、模块、导出规则、错误处理。
2. 结构体、方法、接口、组合。
3. context 的传递与取消。
4. 常见并发基础：goroutine、channel、sync（先会用，不必深挖）。
5. 标准库里和后端最相关的内容：time、net/http、encoding/json、database/sql 基础概念。

### 项目实战任务
- 用最小可运行程序打印配置、启动一个健康检查接口。
- 约定统一错误对象和统一响应结构。

### 验收标准
- 你可以独立写出“接收请求 -> 返回 JSON”最小骨架。
- 你理解 Go 中 error 优先返回的编码风格。

## 阶段 2：HTTP 框架与中间件（3-4 天）
### 学习目标
用 Gin（或你选定框架）搭建规范 HTTP 基座。

### 项目实战任务
1. 建立路由分组：`/api/v1/admin` 与 `/api/v1/public`。
2. 实现基础中间件：request_id、recovery、access log。
3. 打通统一响应结构与错误码返回。
4. 预留 JWT 与 RBAC 中间件挂载点。

### 验收标准
- 路由分组清晰。
- 错误和成功返回结构一致。
- 日志里能追踪请求 ID 与耗时。

## 阶段 3：数据库与迁移（4-6 天）
### 学习目标
掌握 PostgreSQL + GORM（或 sqlx）在企业项目中的基础用法。

### 项目实战任务
1. 按 tech 文档创建迁移脚本目录和版本管理规则。
2. 先落地 3 张关键表：`admin_users`、`vehicles`、`system_settings`。
3. 完成 repository 基础模板：分页查询、事务、软删除。
4. 把索引需求落地到迁移脚本。

### 验收标准
- 迁移可重复执行。
- 能完成基础 CRUD 与分页。
- 能解释为什么某些字段要建索引。

## 阶段 4：认证与权限（2-4 天）
### 学习目标
完成后台登录最小闭环。

### 项目实战任务
1. 实现登录、刷新 token、获取当前用户接口。
2. 完成 JWT 签发/解析与角色注入。
3. 落地 RBAC：超级管理员 vs 运营编辑。
4. 为关键操作加审计日志记录入口。

### 验收标准
- 未登录访问后台接口会被正确拦截。
- 不同角色访问权限符合 PRD。

## 阶段 5：核心业务域实现（8-12 天）
### 学习目标
分域实现 V1 必做功能，形成“可上线的业务骨架”。

### 推荐顺序
1. 分类管理（先做，因为它是车型和模板的前置依赖）。
2. 参数模板管理（定义参数结构）。
3. 车型管理（草稿、发布、下架、复制、预览、批量状态）。
4. 媒体上传签名与回写。
5. 系统设置（价格模式、免责声明、联系方式、默认分享信息）。
6. 公开接口（首页聚合、选车列表、分享落地校验）。

### 每个域都执行同一节奏
1. 先写 domain 规则（状态、约束）。
2. 再写 repository 接口与实现。
3. 再写 service 编排。
4. 最后写 handler 和路由。
5. 同步补单元测试与接口文档。

### 验收标准
- 已上架车型才能被公开接口看到。
- 下架后旧链接返回兜底信息。
- 全局价格模式切换后列表与详情口径一致。

## 阶段 6：缓存、性能与异常兜底（3-5 天）
### 学习目标
让系统达到 PRD 的可用和性能要求。

### 项目实战任务
1. 对首页聚合、分类列表等热点接口加缓存。
2. 设计缓存失效策略（发布/下架/系统设置变更触发）。
3. 完善异常场景返回：无数据、下架、参数为空、联系方式缺失。
4. 增加慢查询日志和接口耗时统计。

### 验收标准
- 接口行为稳定，无明显数据不一致。
- 异常场景都有明确且可读的响应。

## 阶段 7：测试与质量门禁（3-5 天）
### 学习目标
建立企业开发必备的质量控制意识。

### 项目实战任务
1. 为核心规则写单元测试（状态机、价格策略、参数校验）。
2. 为关键接口写集成测试。
3. 为主流程写 e2e（登录 -> 发布 -> 前台可见 -> 下架兜底）。
4. 建立最小 CI 流程（测试、静态检查、构建）。

### 验收标准
- 核心流程有自动化测试兜底。
- 改动后可快速知道是否破坏已有功能。

## 阶段 8：部署与上线准备（2-4 天）
### 学习目标
具备把服务放到测试/生产环境的基本能力。

### 项目实战任务
1. 准备 Docker 镜像与环境变量注入方案。
2. 配置 Nginx 反向代理与 HTTPS。
3. 准备上线检查清单：配置、回滚、监控、日志。
4. 与前端/小程序联调并完成验收项勾选。

### 验收标准
- 测试环境可稳定部署并可回滚。
- 核心监控（成功率、错误率、慢请求）可观测。

## 你每天怎么执行（实操模板）
1. 先选一个“最小功能切片”（例如：分类列表接口）。
2. 写出该切片的输入、输出、业务规则和错误码。
3. 手写实现并本地自测。
4. 补 1-3 个自动化用例。
5. 写一段复盘：今天踩了什么坑，明天先做什么。

## 企业化能力进阶建议
1. 学会写可读性高的业务代码，而不是只追求“能跑”。
2. 任何需求先做边界澄清（必做/不做、同步/异步、一致性级别）。
3. 任何线上问题都形成复盘文档，沉淀到 `server/docs`。
4. 在功能稳定后再做高级优化（并发、分布式、复杂缓存）。

## 最终交付检查清单（对齐 PRD/tech）
1. 接口：管理端与公开端核心 API 全部可用。
2. 数据：核心表结构、索引、迁移脚本完整。
3. 规则：状态机、价格口径、下架兜底、权限边界正确。
4. 质量：单元/集成/e2e 覆盖关键流程。
5. 运维：可部署、可监控、可排障。
6. 文档：接口文档、部署说明、操作说明齐全。

照着这条路线推进，你会从“会写接口”升级为“能独立交付企业后端项目”的工程师。
